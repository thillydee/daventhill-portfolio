When starting the dev server, use background mode:

```
astro dev --background
```

Manage the background server with `astro dev stop`, `astro dev status`, and `astro dev logs`.

## Keeping the downloadable CV in sync

The site offers a one-click CV download (`public/cv/Daven-Thill-CV-{EN,DE}.pdf`, linked from `Header.astro` and `ContactView.astro`). These PDFs are generated from a separate project, `~/Documents/cv-builder` (its own git repo — contains PII like home address and birthdate, so it must never be merged into or pushed alongside this repo).

**Critical**: `~/Documents/cv-builder/data/master/*` is the full CV with home address, birthdate, and marital status — for direct job applications only. The website must only ever get the derived **public** variant (`data/public/*`, generated by `scripts/derive_public_data.js`), which strips all of that. Never copy a file from `cv-builder/output/` to this repo's `public/cv/` unless its filename has the `_Public` suffix.

**Whenever `src/data/profile.ts` or `src/data/profile.de.ts` changes** (new experience bullets, skills, etc.), mirror the same facts into `~/Documents/cv-builder/data/master/data_en.js` and `data_de.js` (adapted to the CV's punchier bullet style — see existing bullets there for the tone), then run the full regenerate workflow documented in `cv-builder/README.md` (re-derive the public variant, regenerate all four docx/PDF, sanity-check the public PDFs contain no address/birthdate/marital-status text, then copy only the `_Public` PDFs here):

```bash
cp ~/Documents/cv-builder/output/Daven_Thill_CV_EN_Public.pdf public/cv/Daven-Thill-CV-EN.pdf
cp ~/Documents/cv-builder/output/Daven_Thill_CV_DE_Public.pdf public/cv/Daven-Thill-CV-DE.pdf
```

Commit both repos separately after.
